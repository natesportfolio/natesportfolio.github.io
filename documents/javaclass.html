For the past one and a half years, I've been teaching an after-school Java programming class to my fellow peers. At first when I got into Java, I tried to learn as much as possible about it. I soon realized that I could also share my knowledge with other students, so I wrote a syllabus and brought it to my principal, which astounded her.



This is the syllabus that I handed to my principal:

Java Class Syllabus

Lesson Structure:

  • 1 hour lessons.
  • The first 10-15 minutes of every class will be spent teaching the students 1-3 different mini-lessons about computer science. This includes subject matter such as the basics of how Java works, how to use Swing, the philosophy of Object-Oriented Programming, etc.
  • The rest of the class will be the students working on projects. This will be interspersed with occasional check-ins by me to see how the students have progressed so far. The students will obviously be free to ask me for help at any time, although this will be discouraged to aid in the student’s inherent problem-solving abilities and debugging skills.

Lesson Plan:

  • Lesson 1:
  • Teach the students how Java works:
  • Navigating Intellij: How to use IntellijIDEA (the text editor for Java) and what most of the buttons do,
  • Creating a project: How to make a project in Intellij, which lets you begin to program,
  • Variables: What a variable is and what their purpose is,
  • Functions: What a function is and why it’s useful,
  • Packages: What a package is and how they help organize programs,
  • Classes: What a class is and how it fits into a program.
  • Teach the basics of how to utilize the console:
  • What the console’s primary function is: How it pertains to debugging, and how it works with console games,
  • Printing/Reading to/from the console: How to print text to the console and how to read a user’s input,
  • How to make a console Hello World program: What a Hello World program is and why use it.
  • Teach code formatting:
  • Where code can be customizable: Where you can put spaces, where you can remove them, where you can put a newline, where you can’t, etc.
  • What function names, variables names and class names should be: What to name variables, functions and classes based on programming naming conventions.
  • What package names should be and how to organize them in accordance with package naming conventions.
  • Lesson 1 will be the students experimenting with console games.
  • Lesson 2:
  • Primitive variables:
  • Booleans: What a boolean is,
  • String: What a string is,
  • Integer: What an integer is,
  • Float: What a floating-point number is.
  • Operators:
  • Conditional Operators: ==, !=, <, >, <=, >=,
  • Arithmetic Operators: +, -, *, /, %, ++, —,
  • Logical Operators: &&, ||, !,
  • Assignment Operators: =, +=, -=, *=, /=, %=,
  • Cast: Explain Casting.
  • Exercise - assign variables.
  • If/Else Statements:
  • What an if/else statement does: How to use if/else statements to do one thing if a condition is true, and another if it’s false,
  • How if/else statements work: How boolean conditionals work, and where to use else if and else,
  • If priorities: What the priorities are for the compiler for if, else if and else.
  • Exercise - use if/else statements.
  • For Statement:
  • What a for statement is: How to loop through possibilities,
  • Where to put a for statement: How to use for statements well in conjunction with arrays and integers,
  • How for statements work well with ifs: How to use the for variable in the for code.
  • Difference between for and while
  • Exercise - use for loop.
  • Lesson 2 will be the students experimenting with objects, variables and conditionals.
  • Lesson 3:
  • Arrays:
  • What an array is: What an array does and how to use it,
  • Where arrays can be useful and an in-depth explanation on how to use them and really customize them,
  • ArrayList vs. []: When to use an ArrayList and when to use the built-in compiler arrays.
  • Methods:
  • Parameters: What parameters are and how to use them,
  • VarArg parameters: What a VarArg parameter is,
  • Why use methods: Why put code into a method.
  • Returning: When to return variables and how.
  • Lesson 3 will be the students experimenting with arrays and OOP.
  • Lesson 4:
  • Classes:
  • Classes: What a class is,
  • Interfaces: What an interface is,
  • Enums: What an enumeration is,
  • Annotations: The @Override annotation.
  • Constructors:
  • Taking parameters: How to use parameters with constructors,
  • Constructor initialization: How to initialize variables inside of constructors.
  • Abstract Classes vs. Interfaces:
  • Difference: What an abstract class is and what an interface is,
  • Why use one or the other: Why use abstract classes and why use interfaces.
  • Teach the basics of Object-Oriented Programming and how to use it effectively:
  • The basics of OOP: What an object is and where to use it,
  • Super/Sub classes: What “extends” means, and how to organize super/sub classes,
  • How to use OOP: When to use primitives and when to use Objects.
  • Lesson 4 will be the students experimenting with different types of classes, methods and abstract classes.
  • Lesson 5:
  • Variables:
  • Initialization vs. abstract: When to initialize and when to leave them abstract,
  • Public Static Final variables: What the point of public static final variables are.
  • Keywords:
  • Static vs. non-static: What static means and when to use it,
  • Final: What final means and when to use it,
  • Public, Private, Protected: What they mean and what the differences are.,
  • Void: What void is.
  • Lesson 5 will be students experimenting with different keywords.
  • Lesson 6:
  • GUIs:
  • How GUIs work in Java: Using Swing to make JFrames,
  • How to display things to GUIs: Using JPanels and Content Panes,
  • How to add content to GUIs: JButton, JLabel, JTextPane, JInputPanes, etc.
  • Doing things with GUIs:
  • Using actions: How to make JButtons do things.
  • Lesson 6 will be students experimenting with GUIs.
  • Lesson 7:
  • Using OSC:
  • Getting started with OSC: What it can do and how to use it,
  • How to use OSC: Splitting into pairs and creating chat applications to communicate with each other,
  • Strict warnings: Tell students not to use chat applications in any other instance than when I’ve allowed it.
  • Lesson 7 will be students splitting into pairs and creating chat applications.
  • Lesson 8:
  • Using images:
  • BufferedImages: Importing BufferedImages from the hard drive,
  • Displaying: Displaying images to screen,
  • Graphics Object: Best way to display images to screen.
  • Lesson 8 will be students experimenting with BufferedImages and starting work on a final project. Project can be anything they desire to create, from a GUI application to a game.
  • Lesson 9:
  • Lesson 9 will be students working on final project.
  • Lesson 10:
  • Lesson 10 will be students working on final project, and at the end presenting if they want.
